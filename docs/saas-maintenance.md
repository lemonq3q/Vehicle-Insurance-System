# SaaS 夜间维护

## 执行时间

- 调度：每天 `04:00:00`。
- 时区：`Asia/Shanghai`。
- 执行模块：`backend/saas`。
- 执行前进入系统维护模式并等待当前请求结束，任务结束或异常时恢复服务。

## 软删除归档

归档服务通过 `information_schema` 动态识别同时满足以下条件的源表：

1. 源表包含 `deleted` 字段。
2. 数据库中存在对应的 `<源表名>_archive` 表。
3. 源表自身不是归档表。

任务在事务中将 `deleted=1` 的数据按源表与归档表共有字段复制到归档表，随后从源表物理删除。没有软删除字段或没有归档表的表不会参与本任务。

## 套餐生命周期

任务按到期时间处理 `status=1 AND end_at<=NOW()` 的订阅，每个订阅使用独立事务：

1. 已有其他有效订阅时，仅将旧订阅标记为过期。
2. 未开启自动续费时，将订阅标记为过期并停用企业全部成员。
3. 自动续费套餐、钱包不存在或余额不足时，创建 `status=6` 的失败订阅订单，记录失败原因，再执行到期处理。
4. 自动续费成功时，锁定钱包并完成扣款、`AUTO_RENEW` 订单、资金流水、订阅周期更新和成员席位同步。
5. 单个企业失败不会阻断其他企业；未预期异常会在独立事务中创建失败订单并执行到期兜底。

## 成员席位同步

- 当前启用人数超过套餐上限：依次停用出单员、管理员、企业拥有者；同角色优先停用后加入成员。
- 套餐上限增加：依次启用企业拥有者、管理员、出单员；同角色优先启用先加入成员。
- 只自动启用 `status=0` 的成员，不会重新启用待审核或已退出成员。
- 套餐到期会停用当前启用或待审核成员。
