# 车险服务企业数据隔离

## 认证规则

- 车险服务登录查询同时读取 `tenant_user.status`、`tenant_member.status` 和 `tenant_member.enterprise_id`。
- 用户账号或企业成员关系处于停用状态时拒绝登录，统一返回 `403 / 账号未启用`。
- 登录成功后，企业 ID 写入 `LoginUser` 并随单点登录会话保存到 Redis。后续请求由安全上下文恢复该企业 ID。

## SQL 隔离规则

`insruance` 模块通过配置 `insurance.tenant-isolation.enabled=true` 开启 MyBatis-Plus 租户拦截器。拦截器对车险业务表的查询、更新、删除和插入自动追加或填充 `enterprise_id`，业务接口无需接收客户端提交的企业 ID。

隔离范围包括企业成员、商户及商户人员、工单及其报价/支付/核保/物流/文件/车辆资料、保险产品、OCR 记录和系统文件等带 `enterprise_id` 的车险表。平台公共权限、角色、用户主表和商户分类不使用该拦截器。

登录阶段和系统定时任务不存在已认证用户上下文，拦截器会跳过租户条件。登录查询因此显式按当前成员关系取得企业 ID；权限查询显式使用该企业 ID，避免跨企业读取角色权限。

## 写入约束

工单聚合、文件、上下游商户、商户员工和企业成员等业务对象统一从 `EnterpriseContextHolder` 取得企业 ID。缺少企业上下文时直接返回 `403 / 当前账号未关联企业`，不再回退到固定企业 ID。

OCR 属于需要特殊处理的非标准读取：接口只接受已上传文件的 ID，并重新查询当前企业可见的 `sys_file` 记录，不信任客户端回传的 OSS 路径。
